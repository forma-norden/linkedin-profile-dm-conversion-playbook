# linkedin-content-team-ops

Use this skill to operate a lean content team with clear role ownership and
handoff standards.

## Required Inputs

- team structure and available hours
- weekly publishing target
- asset production needs
- tool stack for planning and scheduling

## Recommended Core Roles

- GTM copy operator
- DM or conversation operator
- design or asset operator

## Execution Sequence

1. Define role charter and ownership boundaries.
2. Set workflow stages:
   - ideation
   - drafting
   - asset production
   - review
   - scheduling
3. Define SLA per stage and owner.
4. Define weekly review cadence:
   - performance review
   - iteration decisions
5. Produce escalation rules for missed stages.

## Output Contract

Return:

- role_charter_table
- workflow_stage_map
- SLA_matrix
- weekly_ops_cadence
- escalation_rules

## Failure Conditions

- no owner for a critical stage
- no SLA for publish-critical tasks
- no escalation path for blocked items
