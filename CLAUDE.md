# CLAUDE.md - linkedin-profile-dm-conversion-playbook

This repo defines a LinkedIn profile-to-conversation system for B2B GTM teams.
It converts profile traffic, content engagement, and DM outreach into qualified
pipeline conversations with repeatable workflows.

## Skills in This Repo

| Skill | When to use it |
|-------|---------------|
| `linkedin-icp-and-positioning` | defining ICP and message positioning before profile or DM execution |
| `linkedin-profile-conversion-architecture` | structuring profile headline, about, featured, and CTA placements |
| `linkedin-lead-magnet-post-system` | designing post-level lead magnet flows and response handling |
| `linkedin-dm-sequence-operator` | running first-touch and follow-up DM sequences by trigger type |
| `linkedin-conversation-qualification` | qualifying and routing conversations from DM into next-step actions |
| `linkedin-pipeline-governance` | enforcing safety, compliance, and measurement controls |
| `linkedin-ai-postcraft-workflow` | generating AI-assisted LinkedIn drafts with claim verification |
| `linkedin-content-bucket-planner` | planning weekly content allocation across conversion buckets |
| `linkedin-post-type-composer` | composing lead magnet, case study, mistake, framework, and contrarian posts |
| `linkedin-distribution-loop-operator` | running daily distribution and signal-capture routines |
| `linkedin-content-team-ops` | operating role ownership, SLAs, and production workflow |
| `linkedin-authority-channel-flywheel` | defining positioning and channel expansion tied to pipeline outcomes |

## Output Rule

Every output must include:

- target segment assumptions
- message or workflow artifact
- qualification or routing rule
- clear next action and owner

## Testing Rule

Run tests from the `tests/` folder before production use.
